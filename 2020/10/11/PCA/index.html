1.简介

  1. 主成分分析（Principal component analysis，PCA）是一种常用的无监督学习方法。
  2. PCA是通过正交变换将线性相关的变量转换为少数线性无关的变量，这些线性无关的变量称为主成分。因为主成分数量通常小于原变量的个数，所以PCA属于降维方法。
  3. PCA的常见应用为：通过降维，可以提高算法的运行速度，也可以可视化高维的数据。

2.数学角度

2.1几何意义（李航p298p_{298}p298​）
  • 主成分分析旨在选取正交变换中方差最大的变量，即在旋转变换中选取离样本点距离平方和最小的轴为第一主成分，第二第三同理。
  • 如图，有5个原始数据点，坐标分别为：(2,2),(2,6),(6,2),(6,6),(8,8)(2,2),(2,6),(6,2),(6,6),(8,8)(2,2),(2,6),(6,2),(6,6),(8,8)；
    通过正交变换之后，坐标约等于(−4,0),(−1.1,2.8),(−1.1,−2.8),(1.7,0),(4.5,0)(-4,0),(-1.1,2.8),(-1.1,-2.8),(1.7,0),(4.5,0)(−4,0),(−1.1,2.8),(−1.1,−2.8),(1.7,0),(4.5,0)

  • 1.变量声明：
    设X_train为m×n的样本矩阵(m个样本，n个特征)，其转置为XXX (n*m)，∑\sum∑为XXX的协方差矩阵，∑\sum∑的特征值为λ1≥λ2≥λ3...≥λj≥0λ_1≥λ_2≥λ_3...≥λ_j≥0λ1​≥λ2​≥λ3​...≥λj​≥0，对应的单位特征向量α1,α2,α3...αj\alpha_1,\alpha_2,\alpha_3...\alpha_jα1​,α2​,α3​...αj​。

    XXX的第kkk个主成分为：$$y_k=\alpha_k^TX =\left[\begin{matrix}\alpha_{1k}\\alpha_{2k}\.\.\\alpha_{jk}\end{matrix}\right]^T*\left[\begin{matrix}x_1\x_2\x_3\.\.\x_j\end{matrix}
    \right]=\alpha_{1k}x_1+\alpha_{2k}x_2+…+\alpha_{jk}x_j,k=1,2,…,j$$

其中，x1x_1x1​,x2x_2x2​,x3x_3x3​…均为1*m的向量。

4.PCA代码实现

4.1源码实现

  • PCA的步骤（已有m条n维数据X_train）：

    1.将X_train按列(即属性)进行零均值化，得nor_Xnor\_Xnor_X #shape=(m,n)shape=(m,n)shape=(m,n)；

    2.求协方差矩阵∑=1m−1nor_XTnor_X\sum=\frac{1}{m-1}nor\_X^Tnor\_X∑=m−11​nor_XTnor_X；
    3.求协方差矩阵∑\sum∑的特征值eig_val和对应特征向量eig_vec；
    4.根据特征值降序将对应特征向量从左至右组成矩阵，取前k列组成矩阵PPP #shape=(n,k)shape=(n,k)shape=(n,k)；
    5.降维之后的数据pca_X_train=nor_X×Pnor\_X×Pnor_X×P #shape=(m∗k)shape=(m*k)shape=(m∗k)

    ps:协方差矩阵的无偏估计的系数为1m−1\frac{1}{m-1}m−11​，但无论∑\sum∑的系数不影响最终的特征向量，只影响特征值，因此实际计算时也可以省略1m−1\frac{1}{m-1}m−11​。但np.cov是按照m-1计算的,所以这里我们也用1m−1\frac{1}{m-1}m−11​。

  • 代码：

# X为要降维的数据，类型为DataFrame
def pca(X,k):
    n_samples,n_features=X.shape
    # 零均值化
    nor_X=X-np.mean(X,axis=0)
    # 求协方差矩阵
    C=1/(n_samples-1)*np.dot(np.transpose(nor_X),nor_X)
    # 求特征值和特征向量
    eig_val,eig_vec=np.linalg.eig(C)
    # 组成矩阵，并保留k个向量
    eig_pairs=[(np.abs(eig_val[i]),eig_vec[:,i]) for i in range(n_features)]
    eig_pairs.sort(key=lambda x:x[0],reverse=True)
    feature=np.array([ele[1] for ele in eig_pairs[:k]])
    # 将原数据转换
    data=np.dot(nor_X,np.transpose(feature))
    return data

5.使用PCA的一些建议

5.1优点
  • PCA的参数均通过拟合train来获得，之后在cross_validation或test上用拟合好的PCA映射来完成验证或预测。
  • 使用PCA可以大概把n_features减少1/10甚至1/5，而不影响性能（如accuracy）。
  • PCA的两种主要应用：
    1.Compression（数据压缩→提高算法的速度）
    2.Visualization（降维之后可视化），通常降至2D或3D
5.2PCA的误用
  • 在处理过拟合问题时，虽然使用PCA降维可能会有一定的效果，但是更推荐使用正则项。因为PCA在降维时没有考虑标签列，这样做可能会损失一些重要的信息。而使用正则项时，很容易打到PCA的效果，而且因为J(θ)J(\theta)J(θ)中包含标签列，所以并不会忽略标签列的信息。
  • 在设计机器学习系统时，不应该先把PCA设置为一个步骤。应该先使用原始数据来进行拟合预测。而当有足够的原因，如磁盘不够，运算太太太慢等，才考虑使用PCA。
